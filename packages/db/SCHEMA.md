# Database Schema

Single Postgres instance (Supabase), two schemas:

- **`raw`** — append-only collector output. Source of truth for "what the API actually returned." Never updated in place.
- **`app`** — query-ready, normalized state. What the frontend reads, what engines write.

> Source of truth: [`migrations/0001_init.sql`](./migrations/0001_init.sql). When migrations change, update this file in the same PR.

## Extensions

| Extension  | Why                                     |
| ---------- | --------------------------------------- |
| `vector`   | pgvector — embeddings for T2 (PRD §6)   |
| `pgcrypto` | `gen_random_uuid()` for primary keys    |

## Conventions

- **UUID primary keys** generated by `gen_random_uuid()`.
- **Timestamps** are `timestamptz`, defaulting to `now()`.
- `app.project.updated_at` auto-maintained by trigger `app.set_updated_at()`.
- **Soft references**: `raw.snapshot.project_id` is `on delete set null` so deleting a project doesn't lose its history. Most `app.*` FKs are `on delete cascade`.
- **Enum-like text columns** use `check` constraints (not Postgres enums — easier to evolve via migration).
- **Unique constraints** carry semantic meaning (e.g. `app.project.slug`, `app.identity_link (platform, external_id)`).

## `app` schema

### `app.project`

One row per project. The frontend's main read source.

| Column            | Type          | Notes                                          |
| ----------------- | ------------- | ---------------------------------------------- |
| `id`              | `uuid`        | PK, `default gen_random_uuid()`                |
| `slug`            | `text`        | **unique** — URL-safe identifier               |
| `name`            | `text`        | not null                                       |
| `one_liner`       | `text`        | short description                              |
| `category`        | `text`        | primary language or topic                      |
| `primary_url`     | `text`        | canonical project URL                          |
| `status`          | `text`        | `check in ('active','dead')`, default `active` |
| `seo_title`       | `text`        | for project detail page                        |
| `seo_description` | `text`        |                                                |
| `hero_image_url`  | `text`        |                                                |
| `created_at`      | `timestamptz` | default `now()`                                |
| `updated_at`      | `timestamptz` | maintained by `project_set_updated_at` trigger |

### `app.identity_link`

Maps a project to its presence on each platform. Enables cross-platform tracking (PRD §6).

| Column        | Type           | Notes                                                                |
| ------------- | -------------- | -------------------------------------------------------------------- |
| `id`          | `uuid`         | PK                                                                   |
| `project_id`  | `uuid`         | FK → `app.project(id)` `on delete cascade`                           |
| `platform`    | `text`         | `check in ('github','product_hunt','hacker_news','reddit','x')`      |
| `external_id` | `text`         | the platform's own ID for the project                                |
| `confidence`  | `numeric(3,2)` | 0.0–1.0; how sure we are this link is correct                        |
| `source`      | `text`         | `check in ('hard','soft','embedding','manual')` — how we found match |
| `created_at`  | `timestamptz`  |                                                                      |

Indexes: `(project_id)`. Unique: `(platform, external_id)`.

### `app.project_metric`

Daily time-series. One row per project per day. Powers charts and velocity signals.

| Column                   | Type      | Notes                                |
| ------------------------ | --------- | ------------------------------------ |
| `id`                     | `uuid`    | PK                                   |
| `project_id`             | `uuid`    | FK → `app.project(id)` cascade       |
| `date`                   | `date`    | calendar day this row summarizes     |
| `github_stars`           | `integer` |                                      |
| `github_stars_delta_24h` | `integer` | star change vs. previous day         |
| `ph_upvotes`             | `integer` | Product Hunt                         |
| `ph_rank`                | `integer` |                                      |
| `hn_score`               | `integer` | Hacker News                          |
| `reddit_mentions`        | `integer` |                                      |

Indexes: `(date)`, `(project_id)`. Unique: `(project_id, date)`.

### `app.project_embedding`

pgvector store for T2 cross-platform identity matching.

| Column             | Type           | Notes                                         |
| ------------------ | -------------- | --------------------------------------------- |
| `project_id`       | `uuid`         | **PK** — one embedding per project, cascade   |
| `embedding`        | `vector(1536)` | not null; OpenAI `text-embedding-3-small` dim |
| `source_text_hash` | `text`         | dedupe re-embedding the same input            |
| `model_version`    | `text`         | which embedding model produced this           |
| `created_at`       | `timestamptz`  |                                               |

Index: HNSW on `embedding vector_cosine_ops` for approximate nearest-neighbor.

### `app.signal`

Trending / growth signals — what's worth attention right now. Written by the
rule-based engine `apps/worker/src/scripts/run-signals.ts` (migration 0006
replaced the unused v0.1 digest-era table with this schema).

| Column        | Type          | Notes                                                                                                                              |
| ------------- | ------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `id`          | `uuid`        | PK                                                                                                                               |
| `project_id`  | `uuid`        | FK → `app.project(id)` cascade                                                                                                   |
| `signal_type` | `text`        | `check in ('github_star_burst','hn_wave','ph_launch_hot','youtube_spike','cross_platform_heat','new_discovery','rising_trend')` |
| `severity`    | `int`         | `check between 1 and 5`                                                                                                          |
| `title`       | `text`        | short headline, e.g. `⭐ 340 stars in 24h`                                                                                       |
| `description` | `text`        | one-liner detail                                                                                                                 |
| `metadata`    | `jsonb`       | `default '{}'` — `{delta, timeframe_hours, platform_count, ...}`                                                                  |
| `created_at`  | `timestamptz` | refreshed on each upsert                                                                                                         |
| `expires_at`  | `timestamptz` | auto-cleanup hint; null = no expiry                                                                                             |

Unique `(project_id, signal_type)` — one live signal of each type per project
(the engine upserts). Indexes: `(project_id)`, `(signal_type)`, `(severity desc)`.

### `app.subscriber`

Email subscribers for the daily digest.

| Column           | Type          | Notes                                                |
| ---------------- | ------------- | ---------------------------------------------------- |
| `id`             | `uuid`        | PK                                                   |
| `email`          | `text`        | **unique**                                           |
| `status`         | `text`        | `check in ('active','unsubscribed','bounced')`       |
| `preferences`    | `jsonb`       | freeform: categories of interest, frequency, etc.    |
| `source`         | `text`        | where they signed up                                 |
| `created_at`     | `timestamptz` |                                                      |
| `last_opened_at` | `timestamptz` |                                                      |

### `app.digest_run`

Record of each digest sent to each subscriber. Used for open/click analytics.

| Column                | Type          | Notes                                       |
| --------------------- | ------------- | ------------------------------------------- |
| `id`                  | `uuid`        | PK                                          |
| `subscriber_id`       | `uuid`        | FK → `app.subscriber(id)` cascade           |
| `sent_at`             | `timestamptz` |                                             |
| `included_signal_ids` | `uuid[]`      | which signals went into this send           |
| `opened_at`           | `timestamptz` | null until the recipient opens              |
| `click_count`         | `integer`     | default 0                                   |

Indexes: `(subscriber_id)`, `(sent_at desc)`.

## `raw` schema

### `raw.snapshot`

Append-only raw collector output. One row per fetch per project per platform.

| Column       | Type          | Notes                                                                    |
| ------------ | ------------- | ------------------------------------------------------------------------ |
| `id`         | `uuid`        | PK                                                                       |
| `project_id` | `uuid`        | FK → `app.project(id)` `on delete set null` (preserve history)           |
| `platform`   | `text`        | `check in ('github','product_hunt','hacker_news','reddit','x')`          |
| `timestamp`  | `timestamptz` | when the fetch happened                                                  |
| `stars`      | `integer`     | platform-agnostic numeric fields (filled by whichever ones the API has)  |
| `forks`      | `integer`     |                                                                          |
| `upvotes`    | `integer`     |                                                                          |
| `comments`   | `integer`     |                                                                          |
| `rank`       | `integer`     |                                                                          |
| `raw_data`   | `jsonb`       | not null — the full API payload, queryable via `->`/`->>`                |

Indexes: `(platform, timestamp desc)`, `(project_id)` (partial, where not null), GIN on `raw_data`.

### `raw.collector_error`

Observability for collector runs that failed.

| Column        | Type          | Notes                                          |
| ------------- | ------------- | ---------------------------------------------- |
| `id`          | `uuid`        | PK                                             |
| `platform`    | `text`        |                                                |
| `error_type`  | `text`        | e.g. `store_repo_failed`, `rate_limited`       |
| `payload`     | `jsonb`       | freeform details (failing input, error message)|
| `occurred_at` | `timestamptz` |                                                |

Index: `(platform, occurred_at desc)`.

## Functions

### `app.set_updated_at()`

Trigger function — sets `new.updated_at = now()` before update. Attached to `app.project` as `project_set_updated_at`. Add to other tables as they get an `updated_at` column.

---

## Permissions

`service_role` has full access to both schemas (`grant all` + default-privileges). The worker uses a direct Postgres connection (`postgres.{ref}` user via the Supabase pooler), which has full access to everything since it's the project owner.

## RLS

**Deferred to a later migration.** v0.1 is dogfood-only — the worker writes via direct PG, and there's no public web read path yet. Before going public we add a follow-up migration that enables RLS on `app.project`, `app.signal`, `app.project_metric` with public-read policies. See PRD §11.
