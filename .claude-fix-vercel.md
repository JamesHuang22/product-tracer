# Fix Vercel Deployment for product-tracer

## Goal
Get `https://product-tracer.vercel.app` to reliably serve the Next.js frontend without `Application error: a server-side exception` on page refresh.

## Current Status
- **Repo**: JamesHuang22/product-tracer (monorepo, pnpm)
- **Frontend**: `apps/web/` (Next.js 15 + App Router)
- **Vercel project**: `product-tracer` (team: jameshuangs-projects)
- **Prod URL**: https://product-tracer.vercel.app
- **Working deployment**: 23h ago (commit `5a12625`), deployed manually via Vercel UI
- **New commits on main**: 3 subsequent commits are **blocked** — Vercel shows "Blocked" status, not deploying automatically

## What's broken
1. **Git Integration is not connected** — Vercel shows "1/5 Connect Git Repository" on the Production overview. New pushes to `main` do NOT trigger automatic deployments (they stay "Blocked").
2. **On page refresh**: The site sometimes returns HTTP 200, sometimes HTTP 500 with `Digest: 4164112120` — inconsistent. The error appears to be a Supabase `postgres` connection exhaustion (too many serverless instances opening connections simultaneously).
3. The latest code on main has a `postgres` connection caching fix (commit `76428b7`) but it hasn't been deployed because Git auto-deploy is broken.

## What to do

### Step 1: Fix Git Integration
Connect the Vercel project to GitHub:
- Use `vercel git connect` or visit Vercel Dashboard → Settings → Git → Configure Git Provider
- Connect to `JamesHuang22/product-tracer`
- Verify that new pushes to `main` trigger automatic deployments

### Step 2: If Git Integration can't be fixed, deploy manually
```
cd /Users/jameshuang/.openclaw/workspace/product-tracer
npx vercel --prod --confirm
```

### Step 3: Check Vercel env vars
The Vercel project needs these env vars set (they were apparently pushed once but may have been lost):
- `DATABASE_URL` — Supabase transaction pooler (port 6543), NOT session pooler (port 5432)
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

Verify: `vercel env ls`

### Step 4: Build locally first to confirm it works
```
pnpm install --frozen-lockfile
cp .env.example .env   # James fills in the real values
pnpm --filter @product-tracer/web build
```

### Step 5: Check production logs after deploy
```
npx vercel logs product-tracer.vercel.app
```
Look for the actual error causing the 500 — it should be a `postgres` connection issue.
